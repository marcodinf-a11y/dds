# at-73d6ad76: Spec Schema: Markdown Heading Extractor

## Objective

Implement the shared Markdown heading extractor that all Ring 0 validators use for template structure checks. After this task, a pure function exists that takes a Markdown string and returns an array of heading metadata including level, text, slug, and line boundaries.

## Context

The heading extractor is a shared utility defined in `src/parsers/markdown.ts`. It is consumed by all three document-level Ring 0 validators (spec, impl doc, atomic task) to validate Markdown template structure â€” correct H1 title, required H2 sections in order, non-empty sections.

The TypeScript types from at-e27804a5 are available in `src/types/`. The extractor does not depend on them directly but follows the same naming conventions established there.

The extractor must return objects with the following shape:
```typescript
{ level: number; text: string; slug: string; startLine: number; endLine: number }
```

Where:
- `level`: 1 for H1, 2 for H2, 3 for H3
- `text`: the heading text after stripping `#` characters and leading/trailing whitespace
- `slug`: the text lowercased, spaces replaced with hyphens, special characters (anything not alphanumeric or hyphen) stripped â€” follows standard Markdown anchor rules
- `startLine`: 1-based line number where the heading appears
- `endLine`: 1-based line number of the last line of content before the next heading (or end of file)

No full Markdown AST parser is used. Only ATX-style headings (lines starting with `#`) are recognized. This is a deliberate design decision to avoid heavy dependencies like `remark`/`unified`.

## Approach

1. In `src/parsers/markdown.ts`, define and export an interface `HeadingInfo` with fields `level: number`, `text: string`, `slug: string`, `startLine: number`, `endLine: number`.

2. Implement and export a helper function `generateSlug(text: string): string` that lowercases the input, replaces spaces with hyphens, removes characters that are not alphanumeric or hyphens, and collapses consecutive hyphens into one.

3. Implement and export a function `extractHeadings(markdown: string): HeadingInfo[]`. Split the input on newlines. Iterate through lines (1-based indexing). For each line matching the regex `^(#{1,6})\s+(.+)$`, create a `HeadingInfo` entry with `level` as the count of `#` characters, `text` as the captured group trimmed, `slug` from `generateSlug(text)`, and `startLine` as the current line number.

4. After collecting all headings, compute `endLine` for each heading: it is one less than the `startLine` of the next heading, or the total line count for the last heading. Handle the edge case where the file ends with a newline (do not count trailing empty line as content).

5. Return the array of `HeadingInfo` objects in document order.

## Constraints

- Do not use any external Markdown parsing library (`remark`, `unified`, `markdown-it`, etc.).
- Do not perform file I/O; the function takes a string parameter.
- Do not recognize setext-style headings (underline with `===` or `---`); only ATX-style.
- Do not modify files outside the declared scope.
- Export both `HeadingInfo` and `extractHeadings` as named exports.
