# at-3a7815aa: Impl Schema: Ring 0 Unit Tests and Fixtures

## Objective

After this task, the project has comprehensive unit tests for the impl-doc Ring 0 validator covering all 18 rules (R0-I40 through R0-I51, R0-I60 through R0-I67) and for the graph utility functions (buildAdjacencyList, detectCycles), with synthetic fixture files for valid and invalid impl doc cases.

## Context

**Files:**
- `tests/unit/ring0/impl-validator.test.ts` -- Main test file using vitest.
- `tests/fixtures/impl-docs/valid-impl.json` -- A valid ImplDefinition JSON fixture.
- `tests/fixtures/impl-docs/valid-impl.md` -- A valid impl doc Markdown fixture matching the JSON.
- `tests/fixtures/impl-docs/invalid-impl-samples.ts` -- TypeScript module exporting invalid fixture variations for each rule.

**Testing pattern from impl-7e2a9f1b:** Unit tests use vitest with `describe`/`it` blocks organized by rule. Each rule gets at least one positive test (valid input passes) and one negative test (invalid input triggers the expected failure). Fixtures are synthetic and self-contained.

**What to test:**

Graph utilities (`src/parsers/graph.ts`):
- `buildAdjacencyList`: empty input, single edge, multiple edges, nodes with no outgoing edges.
- `detectCycles`: acyclic graph, self-loop, simple two-node cycle, transitive cycle (A->B->C->A), disconnected graph with one cyclic component.

Ring 0 validator (`src/validators/impl/ring0.ts`):
- R0-I40: Valid JSON passes schema; invalid JSON (missing required field, wrong type) fails.
- R0-I41: Unique ID passes; duplicate ID fails.
- R0-I42: Correct description pattern passes; wrong pattern fails.
- R0-I43: Valid spec_sections format passes; invalid format fails.
- R0-I44: Valid atomic_tasks references pass; invalid reference fails.
- R0-I45: Valid dependencies references pass; invalid reference fails.
- R0-I46: No self-reference passes; self-reference fails.
- R0-I47: Acyclic graph passes; cyclic graph fails.
- R0-I48: Status-array consistency: draft+empty passes, decomposed+non-empty passes, draft+non-empty fails, decomposed+empty fails.
- R0-I50: Correct parent references pass; mismatched parent fails.
- R0-I51: Valid module subset passes; superset violation fails.
- R0-I60: Correct H1 format passes; wrong format fails.
- R0-I61: Correct 7 H2 sections passes; missing/extra/wrong-order fails.
- R0-I62: Non-empty sections pass; empty section fails.
- R0-I63: Correct 3 H3 subsections passes; missing/extra fails.
- R0-I64: Matching H1 ID passes; mismatched ID fails.
- R0-I66: Presence of REQ-XX passes; absence fails.
- R0-I67: REQ with spec reference passes; REQ without reference fails.

**Fixtures:**
- `valid-impl.json`: A complete, valid ImplDefinition with status `draft`, empty atomic_tasks, one dependency, two modules, and valid spec_sections.
- `valid-impl.md`: Matching Markdown with correct H1, all 7 H2 sections, 3 H3 subsections under Decomposition Notes, REQ-01 with spec reference.
- `invalid-impl-samples.ts`: Exports named objects/functions producing invalid variations (e.g., `missingId`, `duplicateId`, `selfDependency`, `cyclicDeps`, `wrongH2Order`, `emptySection`, `missingReqReference`).

## Approach

1. Create `tests/fixtures/impl-docs/valid-impl.json` with a synthetic valid ImplDefinition:
   - `id`: `impl-00000001`
   - `spec_sections`: `["spec-00000001#test-section"]`
   - `description`: `impl-00000001.md`
   - `modules`: `["mod-a", "mod-b"]`
   - `status`: `draft`
   - `atomic_tasks`: `[]`
   - `dependencies`: `["impl-00000002"]`

2. Create `tests/fixtures/impl-docs/valid-impl.md` with synthetic Markdown containing correct H1 (`# impl-00000001: Test Implementation Document`), all 7 H2 sections in order with non-empty content, 3 H3 subsections under Decomposition Notes, and at least one REQ-01 entry with `(from spec-00000001#test-section)`.

3. Create `tests/fixtures/impl-docs/invalid-impl-samples.ts` exporting factory functions or objects for each invalid case. Each export is named descriptively (e.g., `makeMissingId()`, `makeSelfDependency()`, `makeWrongH2Order()`).

4. Create `tests/unit/ring0/impl-validator.test.ts`:
   - Import `validateImplRing0` from `src/validators/impl/ring0.ts`.
   - Import `buildAdjacencyList`, `detectCycles` from `src/parsers/graph.ts`.
   - Import fixtures.
   - Add a `describe('graph utilities')` block with tests for both functions.
   - Add a `describe('validateImplRing0')` block with nested `describe` blocks per rule.
   - For each rule, write at least one `it('passes when ...')` and one `it('fails when ...')` test.
   - Construct `ImplValidationContext` objects inline for each test case as needed.

5. Run `npx vitest run tests/unit/ring0/impl-validator.test.ts` and verify all tests pass.

6. Run `npx vitest run` to verify no regressions in the full suite.

## Constraints

- Do not modify the Ring 0 validator (`src/validators/impl/ring0.ts`) or graph utilities (`src/parsers/graph.ts`). If tests reveal bugs, report them but do not fix in this task.
- All fixtures must be synthetic -- do not reference real project files or depend on project state.
- Do not exceed 4 files in scope.
- Do not add test dependencies beyond vitest (already a project dependency).
- Test file organization must use `describe`/`it` blocks, not flat test functions.
