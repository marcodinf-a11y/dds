# at-d5c5627d: Pipeline: Ring 1 and Ring 2 Runners

## Objective

Implement the Ring 1 and Ring 2 check runners that bridge per-level prompt templates (from Impl Docs 1/2/3) with the Claude CLI wrapper. After this task, the refinement loop can invoke LLM-based semantic and quality checks for any document level by calling a single function that handles prompt assembly, CLI invocation, and result parsing.

## Context

The Claude CLI wrapper (`src/llm/claude-cli.ts` from at-08d20126) already exists and provides `callClaude()`. The per-level prompt template functions are provided by Impl Docs 1 (spec-level), 2 (impl-doc-level), and 3 (task-level) and will be imported here. Each runner prepends a shared system prompt (defined as a constant in the runner module) to the per-level check prompt before invocation.

Ring 1 checks return structured JSON conforming to `Ring1Result`: `{ check, verdict, issues }`. Ring 2 checks return structured JSON conforming to `Ring2Result`: `{ check, dimension, verdict, evidence, summary }`. These result types are defined in `src/types/results.ts` (from Impl Doc 1).

The shared system prompts are static text from docs/04-validation-pipeline.md and are version-controlled as constants in the source, not loaded from external files.

## Approach

1. Create `src/llm/ring1.ts` exporting:
   - A `RING1_SYSTEM_PROMPT` constant containing the shared Ring 1 system prompt text from docs/04.
   - A `runRing1Check(ruleId: string, documentContent: string, level: 'spec' | 'impl' | 'task', config: PipelineConfig)` function that: (a) dispatches to the correct per-level Ring 1 prompt template function based on `level`, (b) prepends `RING1_SYSTEM_PROMPT` to the per-level prompt, (c) calls `callClaude()` with the combined prompt, the Ring1Result JSON schema, and the configured `timeouts.ring1_check_seconds`, (d) returns the parsed `Ring1Result`.

2. Create `src/llm/ring2.ts` exporting:
   - A `RING2_SYSTEM_PROMPT` constant containing the shared Ring 2 system prompt text from docs/04.
   - A `runRing2Check(ruleId: string, documentContent: string, level: 'spec' | 'impl' | 'task', config: PipelineConfig)` function that: (a) dispatches to the correct per-level Ring 2 prompt template function based on `level`, (b) prepends `RING2_SYSTEM_PROMPT` to the per-level prompt, (c) calls `callClaude()` with the combined prompt, the Ring2Result JSON schema, and the configured `timeouts.ring2_check_seconds`, (d) returns the parsed `Ring2Result`.

3. Both modules import `callClaude` and `PipelineConfig` from `../llm/claude-cli.ts`, result types from `../types/results.ts`, and per-level prompt template functions from the appropriate Impl Doc modules.

## Constraints

- Do not inline per-level prompt templates; import them from the per-level modules (Impl Docs 1/2/3).
- System prompts must be constants, not loaded from external files at runtime.
- Do not add any retry logic; that is handled by `callClaude()` in `claude-cli.ts`.
- Each runner handles exactly one ring; do not combine Ring 1 and Ring 2 logic into a single function.
