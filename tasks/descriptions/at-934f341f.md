# at-934f341f: Spec Schema: Ring 0 Validator

## Objective

Implement the spec-level Ring 0 validator that runs all 14 deterministic structural checks (R0-S01 through R0-S14) against a specification's JSON definition and Markdown description. After this task, specification documents can be structurally validated with a single pure-function call.

## Context

The Ring 0 validator lives at `src/validators/spec/ring0.ts`. It imports:
- `SpecDefinition` from `src/types/definitions.ts` (created in at-e27804a5)
- `Ring0Result`, `Ring0RuleResult` from `src/types/results.ts` (created in at-e27804a5)
- `extractHeadings`, `HeadingInfo` from `src/parsers/markdown.ts` (created in at-73d6ad76)
- The JSON Schema from `src/schemas/spec.schema.json` (created in at-e27804a5)

The validator must implement these 14 rules:
- **R0-S01:** JSON conforms to SpecificationDefinition schema (use ajv)
- **R0-S02:** `id` is unique (within the provided context; for single-doc validation this is trivially true — the cross-doc uniqueness check is a pipeline concern)
- **R0-S03:** `description` field matches the filename pattern `spec-[0-9a-f]{8}\.md`
- **R0-S04:** No self-reference in `related_specs` (spec's own id must not appear in its `related_specs` array)
- **R0-S05:** Markdown starts with H1 matching `# {spec-id}: {title}`
- **R0-S06:** Markdown contains exactly six H2 sections
- **R0-S07:** H2 sections are: Overview, Functional Requirements, Non-Functional Requirements, System Constraints, Glossary, Decomposition Guidance (in that order)
- **R0-S08:** No H2 section is empty (each has at least one non-whitespace line of content)
- **R0-S09:** H1 `spec-id` matches the JSON definition's `id` field
- **R0-S10:** Status-array consistency: if status is `draft` or `validated`, `implementation_docs` must be empty or absent; if `decomposed`, must be non-empty
- **R0-S11:** `related_specs` entries match the `spec-[0-9a-f]{8}` pattern
- **R0-S12:** At least one `FR-XX` identifier exists in the Functional Requirements section
- **R0-S13:** All `FR-XX` and `NFR-XX` identifiers are unique across the document
- **R0-S14:** `version` is a positive integer

The function signature is: `validateSpecRing0(spec: SpecDefinition, markdown: string): Ring0Result`. The `Ring0Result.valid` field is true only if all rules pass.

`ajv` v8 must be used for JSON Schema validation. The schema should be compiled once at module level and reused.

## Approach

1. At the top of `src/validators/spec/ring0.ts`, import `Ajv` from `ajv`, the spec schema JSON, the `SpecDefinition` and result types, and `extractHeadings` from the parser.

2. Create a module-level `Ajv` instance and compile the spec schema once: `const ajv = new Ajv(); const validateSchema = ajv.compile(specSchema);`.

3. Implement a helper function `checkRule(rule: string, passed: boolean, message?: string): Ring0RuleResult` that returns a result entry.

4. Implement `validateSpecRing0(spec: SpecDefinition, markdown: string): Ring0Result`. Inside the function, build an array of `Ring0RuleResult` by calling each rule check sequentially:
   - R0-S01: Call `validateSchema(spec)` and check the result. If invalid, include ajv error messages.
   - R0-S02: Always passes in single-document mode (mark as passed with note).
   - R0-S03: Check `spec.description` matches `/^spec-[0-9a-f]{8}\.md$/`.
   - R0-S04: Check `spec.related_specs` does not include `spec.id`.
   - R0-S05: Call `extractHeadings(markdown)`, find the H1, check it matches `{spec.id}: {spec.title}`.
   - R0-S06: Filter headings to level 2, check count equals 6.
   - R0-S07: Check the six H2 texts match the required names in order.
   - R0-S08: For each H2, check that `endLine - startLine > 0` (has content lines).
   - R0-S09: Extract the spec-id from the H1 text and compare to `spec.id`.
   - R0-S10: Check status-array consistency per the rules above.
   - R0-S11: Validate each `related_specs` entry matches the pattern.
   - R0-S12: Search the Functional Requirements section content for `/FR-\d{2}/` and check at least one exists.
   - R0-S13: Collect all FR-XX and NFR-XX identifiers, check for duplicates.
   - R0-S14: Check `spec.version` is a positive integer.

5. Set `Ring0Result.valid` to true only if every result has `passed === true`. Return the result.

## Constraints

- Do not perform any file I/O; the function receives parsed data as parameters.
- Do not call any LLM or external service.
- Do not modify the `SpecDefinition`, `Ring0Result`, or `HeadingInfo` types.
- Do not add new public exports beyond `validateSpecRing0`.
- Do not modify files outside the declared scope.
